CRUD con Laravel

SEMANA 9: DEL 14 DE MARZO AL 20 MARZO

Laravel nos permite crear fácilmente un CRUD simple para manipular datos en la base de datos, para lo cual también debemos tener un modelo, un controlador y las vistas correspondientes.

¿Qué es CRUD?

CRUD (Create, Read, Update, Delete) por sus siglas en Ingles son las cuatro funciones persistentes en una base de datos que todo sistema web debe hacer para el manejo correcto, limpio y completo de la información de la empresa o sitio en la que desee implementarse.

Modelos

El modelo es el encargado de realizar las consultas a la base de datos, dichas consultas serán las que haremos usando el CRUD, es decir, estas contendrán información referente a la extracción de datos, insertarlos, actualizarlos/editarlos o eliminarlos. Es importante recalcar que cada tabla de nuestra base de datos debe tener un modelo para su correcto funcionamiento.

Controladores

Como su nombre lo indica, este apartado esta encargado de controlar y revisar el comportamiento y manejo de una petición proveniente de “Routes”, es decir, de las rutas del proyecto.

Un dato importante que mencionar es que los controladores creados deben registrarse dentro del archivo “web.php”, si no, el proyecto no lo reconocerá y no tendrá funcionalidad. Los datos o arreglos que obtendremos o enviaremos a la vista deben establecerse en la función del controlador, por lo que es necesario llamar al modelo o simplemente importarlo como un objeto.

Vistas

Creadas y manejadas desde la carpeta “resources”, las vistas son aquellas encargadas de mostrar los datos de nuestra base de datos en el formato que nosotros requiramos, esto siempre y cuando se defina en el controlador, ya que, como se mencionaba anteriormente, estos manejan y controlan la información de las vistas.

Nuevos apartados e información, este tema me toco realizar una guía de lo explicado en clase referente al uso y trabajo de Composer y Laravel, para llevar la práctica continua de estos temas sin importar el momento que sea.
